Calibration database

Format and fields of the calibration database

fields present in the database

  1. camera: Camera model used
  2. shot1: Initial shot for which the calibration is valid
  3. shot2: Final shot for which the calibration is valid
  4. xshift: shift in the x direction to align the map
  5. yshift: shift in the y direction to align the map
  6. xscale: scale in the x direction the align the map
  7. yscale: scale in the y direction to align the map
  8. deg: angle to rotate and align the map
  9. cal_type: Type of calibration [PIX or IMAGE]
  10. diag_ID: Number identifying which fild/inpa/i-HIBP is the calibration for

struc_database

  1. Lines with comments. In principle the default number is 5, but this number can be changes when we read the database, ie, any number can be valid as far as this number is passed as input when the database object is initialised
  2. Database, organised in rows, each row containing all data from one calibration, starting from a calibration ID and followed by all the fields in the order indicated above. See any example for more information
